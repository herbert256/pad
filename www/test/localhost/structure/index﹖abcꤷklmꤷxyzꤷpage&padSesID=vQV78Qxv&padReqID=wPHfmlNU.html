Sequence of calls

Note that some values 'root/abc/klm/xyz' look incorrect. This comes because all inits/exits/page PHP sources are called before the rendering of the PAD templates start. The sequence is.

  • /_inits.php
  • /abc/_inits.php
  • /abc/klm/_inits.php
  • /abc/klm/xyz/_inits.php
  • /abc/klm/xyz/page.php
  • /abc/klm/xyz/_exits.php
  • /abc/klm/_exits.php
  • /abc/_exits.php
  • /_exits.php

So it is possible (and in this example application it happens) that:
- <page>.php PHP code overwrites variables from the _inits.php PHP code
- the _exit.php files overwrites variabeles from <page>.php and _inits.php fPHP code

An earlier version of PAD did prevent this, but in reality it is handy, this way you can set for exampe a $title variable in the page PHP code that is used in the _inits.pad template.
