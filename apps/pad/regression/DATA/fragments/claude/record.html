The record tag executes a custom SQL query.

Syntax 

{record 'SQL query'}
  {$fieldname}
{/record}

Basic Query 

{record 'SELECT * FROM users WHERE active=1'}
  <li>{$name}</li>
{/record}

Executes the SQL and iterates over results.

Join Query 

{record 'SELECT u.name, o.total FROM users u JOIN orders o ON u.id=o.user_id'}
  {$name}: ${$total}
{/record}

Execute join queries to combine data from multiple tables.

Aggregate Query 

{record 'SELECT COUNT(*) as count, SUM(total) as sum FROM orders'}
  Orders: {$count}, Total: ${$sum}
{/record}

Execute aggregate functions like COUNT, SUM, AVG.

With Variables 

{record 'SELECT * FROM orders WHERE user_id=$userId'}
  Order #{$id}: ${$total}
{/record}

Use PAD variables in SQL queries.

Example: Complex Report 

{record 'SELECT
    p.name as product,
    SUM(oi.quantity) as sold,
    SUM(oi.quantity * oi.price) as revenue
  FROM order_items oi
  JOIN products p ON oi.product_id = p.id
  GROUP BY p.id
  ORDER BY revenue DESC
  LIMIT 10'}
  <tr>
    <td>{$product}</td>
    <td>{$sold}</td>
    <td>${$revenue}</td>
  </tr>
{/record}

Comparison: field vs array vs record

Tag Returns Use Case
field Single value Lookup one field
array Table rows Simple table queries
record Query results Custom SQL, joins, aggregates

Security Note

Be careful with SQL injection when using variables in queries. PAD handles basic escaping, but always validate user input.

Note

Database queries require a configured database connection. This demo page does not execute live queries.
