<hr> <h2>Sequence of calls</h2>

<p>Note that some values 'root/abc/klm/xyz' look incorrect. This comes because all inits/exits/page PHP sources are called before the rendering of the PAD templates start. The sequence is.</p>

<ul>
 <li> /_inits.php
 <li> /abc/_inits.php
 <li> /abc/klm/_inits.php
 <li> /abc/klm/xyz/_inits.php
 <li> /abc/klm/xyz/page.php
 <li> /abc/klm/xyz/_exits.php
 <li> /abc/klm/_exits.php
 <li> /abc/_exits.php
 <li> /_exits.php
</ul>

<p>So it is possible (and in this example application it happens) that:<br>
- <page>.php PHP code overwrites variables from the _inits.php PHP code <br>
- the _exit.php files overwrites variabeles from <page>.php and _inits.php fPHP code <br>
<br> An earlier version of PAD did prevent this, but in reality it is handy, this way you can set for exampe a $title variable in the page PHP code that is used in the _inits.pad template.
