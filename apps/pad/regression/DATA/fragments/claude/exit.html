The exit tag terminates PAD processing.

Syntax 

{exit}

What It Does

The exit tag immediately stops PAD processing by calling padExit(). Any content after the exit tag will not be rendered.

Example Usage 

{if $accessDenied eq TRUE}
  <p>Access denied.</p>
  {exit}
{/if}

<p>This content only shows if access is granted.</p>

When to Use

  • Early termination after access denied
  • Stop processing after critical error display
  • End processing after redirect
  • Terminate after API response sent

Comparison with Other Tags

Tag Behavior
exit Stops PAD, outputs current content
error Triggers error handling (may continue)
exception Throws PHP exception
redirect Sends redirect header, then exits

Example: Authentication Guard 

{if $isLoggedIn ne TRUE}
  <h1>Please Log In</h1>
  <p>You must be logged in to view this page.</p>
  <a href="?login">Go to Login</a>
  {exit}
{/if}

<!-- Protected content below -->
<h1>Welcome, {$username}!</h1>

Output Before Exit

Content generated before the exit tag is still output. Use this to display error messages or status information before terminating.

Warning

This demo page does not execute an actual exit. In real usage, the exit tag would immediately terminate PAD processing.
